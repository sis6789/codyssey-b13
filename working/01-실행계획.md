# Part 1. 워크플로우 제안 3종

## 제안 1 (비교구현용 A) — 신규 문의 이메일 자동 분류·배정

**반복 업무**: 팀 대표 메일함으로 들어오는 문의 메일을 사람이 매번 열어보고, 긴급 여부를 판단해 담당자에게 전달하고, 이력을 시트에 남기는 작업.

|구성 요소|내용|
|---|---|
|Trigger|Gmail(또는 Email) – 새 메일 수신 (특정 라벨/제목 키워드)|
|Router/Filter|제목·본문에 "긴급/장애/환불" 키워드 포함 여부로 2분기|
|Action 1|Google Sheets – 문의 이력 Row 추가 (분기 결과 공통 실행)|
|Action 2|Slack(또는 Discord) – 분기별 채널/멘션으로 알림 발송|

**no-code 구현 방안**

1. Make와 Zapier 두 곳에 각각 동일 구조로 시나리오 생성.
2. Trigger는 두 도구 모두 Gmail 커넥터의 "새 이메일" 트리거 사용 (Make: Watch Emails / Zapier: New Email Matching Search).
3. Router(Make) / Filter+분기용 Zap 2개(Zapier, Filter가 단일 분기만 지원하므로 긴급용 Zap과 일반용 Zap을 나눠 동일 Trigger를 각각 연결)로 조건 구성.
4. 두 분기 모두 Google Sheets Add Row 액션 연결 → 이력 공통 기록.
5. 분기별 Slack 채널을 다르게 지정해 Send Message 액션 연결.
6. 테스트 메일 2건(긴급 키워드 포함/미포함)을 보내 두 분기 모두 실행 로그 확인.
7. 계정 리스크 완화: 미션 권장 조합(Google Sheets + Email + Slack) 그대로 사용, 무료 플랜 범위(Make 1,000 Ops/월, Zapier 100 Tasks/월) 내에서 완결 가능.

## 제안 2 (비교구현용 B) — 설문 응답 점수 기반 합격/불합격 처리

**반복 업무**: 구글 폼으로 들어오는 지원자/평가 설문 응답을 열어 점수를 확인하고, 기준 점수 이상인 사람만 별도로 시트에 옮기고 담당자에게 알리는 작업.

|구성 요소|내용|
|---|---|
|Trigger|Google Forms – 새 응답 제출|
|Router/Filter|점수 80점 이상 / 미만 2분기|
|Action 1|Google Sheets – 분기별 다른 시트(합격자/불합격자)에 Row 추가|
|Action 2|Slack – 합격자 발생 시 담당자 멘션 알림|

**no-code 구현 방안**

1. Make: Google Forms Watch Responses → Router(점수 조건 2개 경로) → 각 경로에 Google Sheets Add a Row → 합격 경로에만 Slack Send a Message 추가.
2. Zapier: Google Forms New Response Trigger → Filter by Zapier(80점 이상만 통과하는 Zap)와 별도 Zap(미만만 통과)으로 동일 구조 복제.
3. 테스트 응답을 80점 이상 1건, 미만 1건 제출해 두 분기 모두 실행 확인 및 로그 캡처.
4. 무료 플랜 내 구현 가능(Google Forms/Sheets/Slack 모두 무료 연동), 유료 결제 불필요.

> 제안 1과 제안 2 중 어느 쪽을 프로젝트 1(도구 비교)로 선택해도 미션 요구사항(Trigger 1개 이상, Action 2개 이상, 조건분기 1개 이상, 분기별 1회 이상 실행 확인)을 모두 충족합니다. 아래 Part 2 보고서 샘플은 **제안 1**을 기준으로 작성했습니다. 제안 2로 진행할 경우 도구/워크플로우명만 교체하면 됩니다.

## 제안 3 (나만의 워크플로우, 자유주제·프로젝트 2) — 부정 리뷰 실시간 감지 및 AI 요약 알림

**반복 업무**: 고객 리뷰(구글폼/시트로 수집되는 후기)를 매번 사람이 읽고, 부정적인 내용인지 판단해 요약하고 담당자에게 긴급 공유하는 작업.

|구성 요소|내용|
|---|---|
|Trigger|Google Sheets(또는 Forms) – 새 리뷰 행 추가 감지|
|Router/Filter|별점 3점 이하(부정) / 4점 이상(긍정) 2분기|
|Action 1 (AI, 보너스1)|ChatGPT/Claude – 부정 리뷰 본문을 3줄 요약 + 감정 키워드 추출|
|Action 2|Slack – 부정 리뷰는 요약과 함께 담당자 멘션 긴급 알림 / 긍정 리뷰는 아카이브 채널에 조용히 기록|
|실패 알림(보너스2)|시나리오 실행 실패 시 이메일로 담당자에게 오류 알림, Sheets 기록 실패 시 임시 시트에 대체 저장|

**선정 도구**: Make (시각적 Router 분기 설계가 직관적이고, OpenAI/Anthropic 모듈과 Error Handler를 기본 제공해 보너스 과제까지 하나의 시나리오 안에서 구현 가능하며 무료 Ops 범위 내 완결 가능)

**no-code 구현 방안**

1. Make에서 Google Sheets Watch Rows(신규 리뷰 행)를 Trigger로 연결.
2. Router로 별점 조건 2분기(≤3점 / ≥4점) 구성.
3. 부정 리뷰 경로에 OpenAI(ChatGPT) 또는 Anthropic(Claude) 모듈 연결 → "다음 리뷰를 3줄로 요약하고 감정 키워드를 뽑아줘" 프롬프트로 Action 실행.
4. 요약 결과를 Slack Send a Message 액션에 변수로 삽입해 담당자 채널에 멘션 알림.
5. 긍정 리뷰 경로는 별도 Slack 아카이브 채널로 단순 기록.
6. 시나리오 최상단에 Error Handler(라우트)를 추가해 실행 실패 시 Gmail로 오류 알림 발송, Google Sheets 기록 실패 시 "임시 백업 시트"에 Add Row로 대체 저장.
7. 별점 2점짜리, 5점짜리 테스트 리뷰를 각 1건씩 입력해 두 분기와 AI 요약, 실패 알림 로직까지 실행 로그로 확인.
8. Trigger 발생 시 자동 실행되므로 "실제 동작 워크플로우" 요건과 프로젝트 2의 "Trigger 발생 시 자동 실행" 제약을 충족.
